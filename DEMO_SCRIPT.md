## 🎯 TECHNICAL TALKING POINTS

### When Asked "Tell us about your architecture":
```
1. Presentation Layer: JSP with JSTL for dynamic rendering
2. Controller Layer: Spring MVC @Controller classes
3. Service Layer: @Service classes with @Transactional
4. Data Access Layer: Hibernate DAOs with HQL/Criteria queries
5. Entity Layer: JPA @Entity classes with relationships
6. Database Layer: MySQL with proper indexing & constraints
```

### When Asked "How do you handle complex relationships":
```
"We use Hibernate's sophisticated ORM mapping:
- @ManyToOne for citizen→user relationships
- Foreign keys enforce referential integrity at database level
- Lazy loading prevents N+1 queries
- Transaction management ensures ACID compliance"
```

### When Asked "Why use Hibernate instead of raw SQL":
```
"Hibernate provides:
- Database agnostic code (easy to switch DB)
- Type safety with entity objects
- Automatic schema updates with hbm2ddl
- Relationship management without JOIN queries
- Native SQL fallback for complex reports"
```

## 🎤 ANSWER TEMPLATE FOR COMMON QUESTIONS

**Q: "What's the biggest technical challenge you faced?"**  
A: "Handling complex SQL relationships with Hibernate while maintaining performance. We solved it using lazy loading and proper join fetch strategies in our DAO queries."

**Q: "How do you ensure data consistency?"**  
A: "Through transactional boundaries (@Transactional), foreign key constraints in MySQL, and Hibernate's session-level cache management."

**Q: "What about security?"**  
A: "We use role-based access control with Spring's @WebFilter, SHA-256 password hashing, and secure session management via HttpSession."

**Q: "Why Spring MVC and not Spring Boot?"**  
A: "This was built with Spring MVC + Tomcat to demonstrate classic Java EE architecture including XML configuration and servlet concepts, which are fundamental to enterprise Java."
